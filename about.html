About PerfX

我們的使命：讓瀏覽器也能跑出專業等級效能

PerfX 以瀏覽器為主戰場，提供 CPU、GPU、記憶體吞吐量 (RAM Throughput)、瀏覽器 I/O (Browser I/O)、網路速度 (Network Speed) 的全端壓力測試與意義化對照。

方法論 (Methodology)

  • CPU 多核：Web Worker 多執行緒 + 浮點/矩陣負載，取多輪最大值，套用校準係數對齊 Cinebench R23。
  • CPU 單核：主執行緒延遲敏感負載（分支預測 + 陣列訪問），獨立校準係數對齊單核區間。
  • GPU：WebGL 重度 Fragment Shader + 多次 draw call，若支援 EXT_disjoint_timer_query 則解除 V-Sync 限制。
  • RAM Throughput：大尺寸 Uint8Array 複製，延長時間避免快取偏差。
  • Browser I/O：IndexedDB 寫讀大區塊，檢測高讀速時提示可能為快取。
  • Network Speed：Cloudflare 端點測試延遲與下載吞吐，並顯示 Server: Cloudflare CDN 標註。

校準與對照 (Calibration & Baseline)

  • CPU 基準 100%：Apple M3 / Core i5-13600K ≈ 23,000 分。
  • GPU 基準 100%：RTX 4060 / RTX 3060 Ti ≈ 10,600 分。
  • HARDWARE_TIERS：根據 Cinebench R23 / 3DMark Time Spy 2025 數據，從 Godlike 到 Entry 共多層級。
  • 最近分數使用「最接近分數」匹配，降低 tier 跳動，並顯示等級描述。

資料與隱私 (Data & Privacy)

歷史紀錄與本地排行榜全部儲存在瀏覽器 localStorage；不會上傳伺服器。匯出 JSON/PNG 皆在本地端完成。

技術棧 (Tech Stack)

前端：HTML5 + Tailwind CSS。GPU：WebGL (EXT_disjoint_timer_query)。CPU：Web Worker。圖表：Chart.js。匯出：html2canvas。

使用建議 (Best Practices)

  • 保持裝置接電、性能模式；關閉省電或省電 GPU。
  • 避免背景重載；GPU 測試時請保持頁籤活躍。
  • Browser I/O 若讀速異常高，可能是系統快取。

常見問題 (FAQ)

  • 為何 GPU 分數會因瀏覽器/驅動差異而變動？
    瀏覽器實作與驅動對 WebGL 最佳化不同，且是否支援 EXT_disjoint_timer_query 會影響 FPS 量測。
  • CPU 單核與多核為何不同？
    單核偏重延遲敏感與分支預測，多核偏重吞吐並使用多執行緒。
  • 分數與 Cinebench/Time Spy 一致嗎？
    我們透過校準係數對齊基準，但瀏覽器沙箱與散熱、電源策略仍會帶來差異，請視為近似對照。
